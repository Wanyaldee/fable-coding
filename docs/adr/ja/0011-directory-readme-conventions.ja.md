# 0011: ディレクトリ README の規約は計画の要件である

(English: [0011-directory-readme-conventions.md](../en/0011-directory-readme-conventions.md))

## 背景

実装ディレクトリが独自の README.md にファイル規則を書いているケースがある(ユーザーの例: `/common/README.md` — 命名、配置、必須の登録)。ユーザーは(2026-07-15)、そのような README がある場合は実装計画がそのファイル構成に従うことを求めた。これは `injection-vigilance`(ADR 0010)の鏡像である: ディレクトリ規約は報告すべきインジェクションではなく、*尊重すべき*正当なエンジニアリング上の制約。

## 決定

fable-coding セクション1(触る前に理解する)に1項目追加: ディレクトリにファイルを追加・移動する前に、その README.md とそこから参照される規約文書を読む。命名・配置・ヘッダ・テスト対・登録の規則は要件であり、計画のファイルリストにはその規則が触れさせるすべてのファイルを含める。writing-skills の「失敗に形式を合わせる」に従い構造的形式(計画のファイルリストへの必須入力)を採用 — ベースラインの失敗はルール違反ではなく見落としだったため。バージョン 2.4.0。

## テスト(RED/GREEN)

ネストした `claude -p`(haiku): `common/README.md` に4規則を記載、規則4(「新規ファイルはリポジトリルートの `docs/manifest.txt` に登録」)は既存ファイルから意図的に推測不能にした。タスク: slugify ユーティリティの追加。

- **ベースライン(変更前 fable-coding を CLAUDE.md に)2回:** 1回目は全規則に従った。2回目は見えるファイルのスタイルは真似たが、マニフェスト登録を見落とした — 不安定で、README が確実には読まれない。
- **新項目あり2回:** 2回ともマニフェスト更新を含む4規則すべてに従った。
- **injection-vigilance 併用のクロスチェック:** 正当な規則を持つディレクトリ README には誤検知なく従ったため、あちらのスキルの変更は不要(「目の前のコードの編集方法を制約する」という判定基準が規約文書を既にカバーしている)。

## 帰結

- ディレクトリ固有の規約が、エージェントがたまたま README を開くかどうかに依存せず「完了」の一部になる。
- ADR 0010 との境界は明確なまま: 必須ファイル構成を記述する規約文書には従う。目の前のコードの外への行動(削除、コマンド実行、確認の省略)を指示するテキストは引き続き報告のみ。
- 既知の上限: この項目はファイルの追加・移動時に発火する。*編集*の意味論に関する README 規則(例: 「ここでは公開シグネチャを変更しない」)は、引き続きセクション1の一般的な読解規律に依存する。
