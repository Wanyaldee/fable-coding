# 0004: @armadillo_ai のエージェント規律7則をコーディングスキルに織り込む

(English: [0004-armadillo-seven-rules.md](../en/0004-armadillo-seven-rules.md))

## Context(背景)

ユーザーから X の投稿(`x.com/armadillo_ai/status/2072909284992786535`)にある
エージェント規律7則の取り込みを依頼された(本人の哲学と合致する部分がある
との指摘付き): (1) 着手前に「完了」を機械的に定義、(2) 複数解釈を勝手に
選ばない、(3) ついで改善の禁止、(4) 「動いた」ではなく証拠付きの「検証した」
を報告、(5) 同じエラーへの修正は2回まで、(6) 完了前に初見のセルフレビュー、
(7) 確信度と3点進捗報告。

## Decision(決定)

独立ブロックではなく、各則が属するセクションへ `skills/fable-coding/SKILL.md`
(v1.5.0)に織り込んだ: 則1–2 → §0(計画)、則5 → §2(診断)、則3 → §3
(最小変更)、則4・6 → §5(検証)、則7 → §7(応答スタイル)。アンチパターン
4件追加。和訳も追随。

既存内容との調整が必要だったのは2点:

- **則2 vs「選んで進む」(§9)。** レベルで分ければ矛盾しない: *ユーザーが
  何を望むか*の曖昧さ(成果物が変わる読み方)→ 列挙+推奨+確認。*どう作るか*
  の互角な選択 → 選んで進む。両方の箇条書きが相互参照するようにした。
- **則7の「確信度・中以下なら確認」vs「集められる情報は聞かずに集める」
  (§9)。** ユーザーにしか解決できない不確実性に限定して採用。収集可能な
  ものは従来どおり収集優先。確信度ラベル自体は無条件に適用。

則3・4は既存の §3/§5 と大きく重複していたため、重複させず先鋭化として統合
(隣の改善は実装せず提案として列挙、報告に証拠そのものを含める、スキップは
理由まで明記)。

## Alternatives rejected(却下した代替案)

- **別スキル化** — スコープが誤り: 7則すべてが通常のコーディングタスクで
  発火すべき内容であり、それはまさに `fable-coding` の発火条件。
- **投稿の逐語ブロック引用** — 7則は既存セクションの言い換えまたは先鋭化で
  あり、並行ブロックは1つのスキル内に競合する2つの正を作る。

## Consequences(帰結)

- スキルに完了定義ゲート、修正2回失敗で止まるツーストライク則、懐疑的
  セルフレビューの義務が加わった — 真に新規なのはこの3つの機構。
- ユーザーの dev-philosophy と整合(ヒューマンインザループ: 解釈確認と
  確信度ゲートの質問はその具体化)。
- 天井: ツーストライク則の「同じエラー」の同一性判定は非形式的で、モデルの
  判断に依る。ソースはサードパーティ投稿のスナップショット。出所は本 ADR に
  記録。
